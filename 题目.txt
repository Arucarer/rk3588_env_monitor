

项目名称：RK3568 Linux 智能环境综合监测系统
 
项目描述
 
基于瑞芯微RK3568嵌入式Linux平台，搭建一套多传感器、多协议、云端+本地+手机端三端联动的工业级环境综合监测系统。
设备搭载温湿度、臭氧、PM2.5/PM10大气污染、雨量、土壤PH酸碱度等多路环境传感器。
底层基于Linux内核完成设备驱动适配与设备树开发，中间层基于Modbus RTU完成传感器数据采集，应用层基于Qt开发本地可视化上位机，结合SQLite本地数据库实现数据存储，通过MQTT协议将所有环境数据实时上传云端，支持手机小程序远程查看。
系统集成完整OTA差分升级系统，支持设备远程静默升级、版本回滚、断点续传，实现一套完整闭环的嵌入式Linux工业物联网终端产品。
 
技术栈
 
RK3568、Linux Kernel、设备树DTS、字符设备驱动、Modbus RTU、Qt5 GUI、多线程、SQLite数据库、MQTT物联网协议、云端部署、手机端数据展示、HTTP OTA差分升级、交叉编译、Shell脚本、系统自启动
 
硬件传感器列表（全部纳入项目采集体系）
 
1. 温湿度传感器（I2C）
2. 臭氧浓度传感器（Modbus）
3. PM2.5/PM10大气粉尘污染传感器（Modbus）
4. 雨量降水传感器（脉冲计数/IO中断）
5. 土壤PH酸碱度传感器（模拟量ADC）
6. 大气压力传感器（I2C）
 
项目职责
 
1. Linux底层驱动与设备树开发（核心底层能力）
 
1. 基于RK3568 Linux内核，修改设备树DTS，完成I2C、ADC、UART、GPIO中断资源硬件匹配。
2. 独立开发、调试 ADC模拟量采集驱动，适配土壤PH传感器电压采集，完成滤波、去抖动、数值校准。
3. 开发 GPIO中断驱动，实现雨量传感器脉冲计数，精准统计单位时间降雨量。
4. 适配I2C设备驱动，完成温湿度、气压传感器内核层数据读取。
5. 优化串口驱动时序，解决Modbus总线波特率偏差、帧丢失、干扰问题，保证多传感器总线稳定挂载。
 
2. Modbus总线多传感器数据采集框架
 
1. 在Linux应用层基于串口封装标准Modbus RTU主机协议栈，统一轮询臭氧、大气粉尘等总线设备。
2. 设计总线状态机机制，处理CRC校验、帧超时、设备离线检测、总线异常恢复。
3. 实现多路传感器分时采集、数据去重、滤波校准，避免多设备总线抢占冲突。
 
3. Qt Linux上位机可视化系统（本地终端）
 
1. 基于Qt多线程架构，采集线程、数据库线程、MQTT上传线程、UI线程完全解耦。
2. Qt界面实时展示：温湿度、臭氧、PM2.5/PM10、降雨量、土壤PH、气压全部环境参数。
3. 绘制动态数据曲线、历史趋势图、设备状态指示灯、传感器离线告警提示。
4. 实现界面参数自动刷新、超限变色告警、日志弹窗提示功能。
 
4. SQLite本地数据库存储系统
 
1. 搭建SQLite本地数据库，创建设备参数表、历史环境数据表、告警记录表、系统日志表。
2. 实现定时入库、异常瞬时入库、断电保护缓存写入。
3. 支持Qt端历史数据查询、时间段检索、数据导出、日志回溯分析。
 
5. MQTT云端上传 + 手机端数据展示
 
1. 移植MQTT客户端至RK3568 Linux，配置设备唯一Topic、订阅与发布主题。
2. 所有传感器数据秒级上传云端服务器，JSON格式标准化上报。
3. 云端解析数据并推送至手机端，实现手机远程实时查看所有环境监测参数、历史曲线、设备在线状态、异常告警消息推送。
 
6. 完整Linux端OTA远程升级系统（项目高阶亮点）
 
1. 基于HTTP协议搭建差分OTA升级方案，减少升级包体积，避免全量升级耗时。
2. 开发版本校验、MD5校验、断点续传、升级失败自动回滚机制。
3. 实现后台静默升级，升级过程不影响传感器正常采集业务。
4. 增加升级锁、防重复升级、版本分级管理，保证工业设备升级安全可靠。
 
7. 系统工程化部署
 
1. 编写Shell脚本，实现程序开机自启动、日志轮转、异常重启守护。
2. 优化系统内存调度、线程优先级、IO读写策略，保证7×24小时稳定运行。
3. 完善设备离线、传感器断线、总线异常、网络断连容错机制。
 
项目成果

2. 设备可稳定采集8类环境参数，数据精度高、抗干扰强，支持本地+云端双存储备份。
3. OTA差分升级成功率100%，支持版本回滚，解决工业设备现场维护难题。
4. 整机长时间连续运行无死机、无丢数，达到工业物联网终端量产标准。
